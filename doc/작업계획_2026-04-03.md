## 3. 작업 계획 (우선순위별)

### Phase 1 — 이카운트 대체 최소 기능 (약 4주)

#### P0-1. 매입 관리 신규 구축
- **신규 DB 테이블**
  - `suppliers` — 공급업체 (이름, 연락처, 이메일, 결제 조건)
  - `purchase_orders` — 발주서 (공급업체, 지점, 상태, 총액)
  - `purchase_order_items` — 발주 항목 (제품, 수량, 단가, 입고수량)
- **화면**
  - `/purchases` — 발주서 목록/상세
  - `/purchases/suppliers` — 공급업체 관리
  - 입고 처리 → `inventories` 자동 증가 + `inventory_movements(IN)` 기록
- **대시보드 연동** — 이번 달 매입액 위젯 추가

#### P0-2. 반품/환불 처리
- `sales_orders` 상태에 `REFUNDED` 추가
- 환불 처리 시 재고 역복원 + 포인트 복원 로직
- POS에 환불 버튼 및 환불 전표 생성

#### P0-3. 고객 구매이력 UI
- 고객 상세 페이지에 구매이력 탭 추가
- `sales_orders` JOIN 으로 온·오프라인 통합 표시
- 누적 구매액, 최근 구매일, 구매 빈도 표시

---

### Phase 2 — 회계·운영 강화 (약 4주)

#### P1-1. 기초 회계 인프라
- **신규 테이블**
  - `gl_accounts` — 계정과목 (매출, 매입, 재고자산, 미수금, 미지급금)
  - `journal_entries` — 분개 헤더
  - `journal_entry_lines` — 분개 라인 (차변/대변)
- 판매 완료 시 자동 분개 생성 (매출 인식)
- 입고 완료 시 자동 분개 생성 (매입 인식)
- **화면** — 매출원장, 매입원장, 간이 손익 보고서

#### P1-2. 보안 강화
- RLS 정책 재정의 (역할·지점 기반)
  ```sql
  -- 예: BRANCH_STAFF는 자신의 branch_id 데이터만 조회
  USING (branch_id = auth.jwt() ->> 'branch_id')
  ```
- 서버 액션 (`actions.ts`) 내 세션 토큰 검증 + 역할 체크 추가
- `audit_logs` 테이블 신규 생성 — 누가 언제 무엇을 변경했는지

#### P1-3. 알림 발송 API 연동
- 카카오 비즈니스채널 API 또는 솔라피(Solapi) SMS 연동
- 현재 DB 기록만 → 실제 발송으로 전환
- 발송 결과 Webhook 수신 처리

---

### Phase 3 — 고도화 (약 4주)

#### P2-1. 손익 계산 고도화
- 제품별 원가율·마진율 자동 계산
- 보고서에 매출총이익(GPM) 추가
- 월별 손익 트렌드 차트

#### P2-2. 재고 실사 워크플로우
- 일괄 실사 입력 화면 (현재 시스템 재고 vs 실사 수량)
- 차이 분석 표시 후 일괄 조정 적용
- `inventory_movements(ADJUST)` 자동 생성

#### P2-3. 고객 분석
- LTV(고객생애가치) 계산
- 재구매 주기 분석
- 등급 자동 상향 로직 (누적 구매액 기준)
- VIP/VVIP 타겟 마케팅 발송 자동화

#### P2-4. 고객용 앱 (customer route group)
- `src/app/(customer)/` 신규 route group
- 인증: Cafe24 OAuth + 전화번호 인증
- 화면: 통합 구매내역(온·오프라인), 포인트, 내 정보
